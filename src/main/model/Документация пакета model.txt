Документация пакета model
Общая информация
Пакет main.model содержит три класса для представления графа: Vertex (вершина), Edge (ребро) и Graph (граф). Граф ориентированный. Это значит, что ребро A-B позволяет двигаться только от A к B, но не наоборот. Для двусторонней связи нужно явно создать два ребра: от A к B и от B к A. При создании встречного ребра вес автоматически синхронизируется — если у ребра A-B вес 5, то встречное B-A тоже получит вес 5.

Все классы лежат в пакете main.model. Для использования нужно импортировать их: import main.model.Vertex;, import main.model.Edge;, import main.model.Graph;.

Класс Vertex
Класс представляет вершину графа. Это простая структура данных без сложной логики.

Поля класса:

int id — уникальный идентификатор вершины. Используется для сравнения вершин и поиска.

String name — название вершины, которое отображается на экране визуализатором.

double x — координата X для отрисовки на холсте.

double y — координата Y для отрисовки на холсте.

Конструктор принимает четыре параметра: Vertex(int id, String name, double x, double y). При создании вершины нужно задать все четыре значения.

Геттеры возвращают значения полей: getId() возвращает id, getName() возвращает название, getX() и getY() возвращают координаты.

Сеттеры позволяют изменить координаты: setX(double x) и setY(double y). Это используется визуализатором для перетаскивания вершин мышью.

Для сравнения вершин используется метод equals(). Две вершины считаются равными, если у них одинаковый id. Это стандартное поведение, унаследованное от Object, поэтому при создании вершин важно давать им уникальные id.

Класс Edge
Класс представляет ориентированное ребро графа. Тоже простая структура данных.

Поля класса:

Vertex from — начальная вершина ребра, откуда оно исходит.

Vertex to — конечная вершина ребра, куда оно приходит.

int weight — вес ребра, целое число.

Конструктор: Edge(Vertex from, Vertex to, int weight). Принимает начальную и конечную вершины и вес.

Геттеры: getFrom() возвращает начальную вершину, getTo() возвращает конечную вершину, getWeight() возвращает вес ребра.

Сеттер setWeight(int weight) позволяет изменить вес существующего ребра. Это используется при синхронизации весов встречных рёбер — если у ребра A-B меняется вес, встречное B-A автоматически получает тот же вес.

Ребро ориентированное. Это значит, что ребро, созданное как new Edge(A, B, 5), позволяет двигаться только от A к B. Чтобы добавить движение от B к A, нужно создать отдельное ребро new Edge(B, A, 5).

Класс Graph
Главный класс, контейнер для хранения всех вершин и рёбер графа. Содержит всю логику управления графом.

Внутреннее устройство
Граф хранит данные в двух структурах:

List<Vertex> vertices — список всех вершин графа в порядке их добавления.

Map<Vertex, List<Edge>> adjacencyMap — карта смежности. Ключом является вершина, значением — список всех исходящих из неё рёбер. Это позволяет быстро получать соседей вершины без перебора всех рёбер графа.

Конструктор Graph() создаёт пустой граф с пустым списком вершин и пустой картой смежности.

Работа с вершинами
Метод addVertex(Vertex v) добавляет вершину в граф. Вершина добавляется в список vertices, и для неё создаётся пустой список в adjacencyMap для хранения будущих рёбер.

Метод removeVertex(Vertex v) удаляет вершину из графа вместе со всеми рёбрами, которые с ней связаны. Это довольно сложная операция. Сначала метод собирает все рёбра во всём графе, которые ссылаются на удаляемую вершину (как from, так и to). Затем удаляет эти рёбра из списков смежности всех вершин. Наконец, удаляет саму вершину из vertices и adjacencyMap. Реализовано безопасно: сначала рёбра собираются в отдельный список, потом удаляются разом — так избегается ошибка ConcurrentModificationException.

Метод getVertices() возвращает список всех вершин графа. Алгоритмист может использовать его для инициализации расстояний (установка бесконечности для всех вершин кроме стартовой).

Метод getVertexById(int id) ищет вершину по её идентификатору. Проходит по списку vertices и сравнивает id. Если вершина найдена — возвращает её, если нет — возвращает null. Пригодится, если алгоритмист хранит id вершин в структурах данных и ему нужно получить сам объект Vertex.

Метод getVertexCount() возвращает количество вершин в графе. Полезно для алгоритма Форда-Беллмана, которому нужно знать количество вершин (алгоритм делает V-1 итерацию).

Работа с рёбрами
Метод addEdge(Vertex from, Vertex to, int weight) добавляет ориентированное ребро от from к to с указанным весом. Метод сначала проверяет, нет ли уже ребра между этими вершинами. Если ребро уже существует, его вес просто обновляется — новое ребро не создаётся. Если ребра нет, создаётся новый объект Edge и добавляется в список смежности вершины from. Это единственное место, куда добавляется ребро — граф ориентированный.

Метод removeEdge(Edge e) удаляет ребро из списка смежности его начальной вершины.

Метод getNeighbors(Vertex v) — самый важный метод для алгоритмиста. Возвращает список всех рёбер, исходящих из вершины v. Именно этот метод нужно вызывать на каждом шаге алгоритма Форда-Беллмана для релаксации рёбер. Благодаря тому, что используется adjacencyMap, метод работает мгновенно — просто возвращает готовый список из карты.

Метод getEdgeBetween(Vertex from, Vertex to) ищет ребро от from к to. Проходит по списку рёбер вершины from и проверяет, совпадает ли to с конечной вершиной. Если ребро найдено — возвращает его, иначе возвращает null. Этот метод используется UI для проверки существования ребра перед созданием нового и для синхронизации весов встречных рёбер.

Очистка
Метод clear() полностью очищает граф: удаляет все вершины из списка vertices и все записи из adjacencyMap. После очистки граф можно использовать заново.

Важные особенности для алгоритмиста
Граф ориентированный. Метод getNeighbors(v) возвращает только те рёбра, у которых from == v. Если граф нужно рассматривать как неориентированный, алгоритмист должен сам позаботиться об этом — либо визуализатор обеспечит создание встречных рёбер, либо алгоритмист должен проверять оба направления.

Для алгоритма Форда-Беллмана алгоритмисту потребуется список всех рёбер графа. Его можно получить, пройдя по всем вершинам через getVertices() и для каждой вершины вызвав getNeighbors(). Собрав все полученные рёбра в один список, алгоритмист получит полный список рёбер для релаксации.

Граф поддерживает отрицательные веса рёбер. Никаких проверок на положительность веса нет — это позволяет использовать алгоритм Форда-Беллмана во всей его полноте, включая обнаружение отрицательных циклов.

Класс Graph не проверяет связность графа. Если стартовая и конечная вершины находятся в разных компонентах связности, алгоритм Форда-Беллмана отработает и вернёт бесконечное расстояние для конечной вершины. Интерпретация этого результата — задача алгоритмиста.

Класс Graph не проверяет, существуют ли вершины, передаваемые в методы addEdge. Если передать вершину, которой нет в графе, adjacencyMap.get(from) вернёт null, и вызов .add(edge) вызовет NullPointerException. Визуализатор гарантирует корректность вызовов, но алгоритмисту стоит помнить об этом при тестировании.
