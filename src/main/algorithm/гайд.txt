Документация пакета algorithm

Пакет algorithm содержит реализацию алгоритма Форда-Беллмана, предназначенного для поиска кратчайших путей в ориентированном графе с поддержкой ребер с отрицательным весом и детекцией отрицательных циклов.

Класс BellmanFordAlgorithm
Основной класс, инкапсулирующий логику вычислений.

Метод: BellmanFordResult execute(Graph graph, Vertex startVertex)
Точка входа для запуска алгоритма. Принимает объект графа (Graph) и стартовую вершину (Vertex). Возвращает объект BellmanFordResult, содержащий историю всех промежуточных состояний и финальные результаты вычислений.

Метод: List<Vertex> getShortestPath(Vertex target, Map<Vertex, Vertex> predecessors, Map<Vertex, Integer> distances)
Метод для восстановления кратчайшего пути от стартовой вершины до целевой (target). Принимает карту предшественников и карту итоговых расстояний. Возвращает упорядоченный список вершин (List<Vertex>), составляющих путь. Если путь не существует (вершина недостижима), метод возвращает пустой список.

Класс BellmanFordResult
Класс-контейнер, агрегирующий итоговые данные работы алгоритма.

Метод: List<BellmanFordStep> getStepsHistory()
Возвращает список всех шагов алгоритма.
Важно: История содержит запись о каждой попытке релаксации ребра (каждая итерация цикла по рёбрам), включая те, что не привели к улучшению расстояния. Это позволяет визуализатору отображать процесс проверки каждого ребра в деталях.
Метод: List<BellmanFordStep> getStepsHistory()
...
Примечание: Каждый объект BellmanFordStep содержит активную вершину (getActiveVertex) и 
полные снимки состояния (getDistancesSnapshot, getPredecessorsSnapshot). 
Визуализатор может использовать эти данные для формирования истории посещений 
вершин и динамики изменений на каждом шаге алгоритма.

Метод: Map<Vertex, Integer> getFinalDistances()
Возвращает финальное соответствие вершин и минимальных расстояний до них. Используется для отрисовки весов на вершинах по окончании работы.

Метод: Map<Vertex, Vertex> getPredecessors()
Возвращает карту предшественников (кто «привел» в вершину). Используется для восстановления маршрутов.

Метод: boolean hasNegativeCycle()
Возвращает true, если был обнаружен отрицательный цикл. Сигнал для визуализатора о невозможности корректного поиска путей.

Метод: Set<Vertex> getUnreachableVertices()
Возвращает множество вершин, недостижимых или «зараженных» отрицательным циклом. Предназначено для подсветки таких вершин в интерфейсе.

Класс BellmanFordStep
Класс, описывающий состояние алгоритма на конкретном шаге релаксации рёбер.

Метод: Edge getProcessedEdge()
Возвращает ребро, которое проходит проверку на текущем шаге.

Метод: Vertex getActiveVertex()
Возвращает вершину, которая является «целью» релаксации (куда пытаемся дойти).

Метод: Map<Vertex, Integer> getDistancesSnapshot()
Возвращает снимок всех расстояний в графе на момент текущего шага.

Метод: Map<Vertex, Vertex> getPredecessorsSnapshot()
Возвращает снимок карты предшественников на момент текущего шага.

Метод: boolean isDistanceUpdated()
Возвращает true, если на данном шаге произошло успешное обновление расстояния.

Метод: String getStepDescription()
Возвращает подробное текстовое описание результата проверки (например: "Итерация 1. Ребро A->B. Путь до B улучшен: 10 -> 5" или "Итерация 1. Ребро A->B. Путь до B не изменен: путь не короче текущего").

Примечание для визуализатора:
- Используйте isDistanceUpdated() для определения стиля анимации каждого шага: true — ребро успешно "расслабило" путь (например, зеленый цвет), false — ребро не привело к улучшениям (например, серый цвет).
- Каждый шаг содержит полный снимок состояния графа (distancesSnapshot и predecessorsSnapshot), что гарантирует корректность отрисовки даже для безуспешных проверок.
