СПЕЦИФИКАЦИЯ

Общее описание Разрабатывается визуализатор алгоритма Форда-Беллмана для поиска кратчайшего пути во взвешенном ориентированном графе. Приложение с графическим интерфейсом на Java (JDK 21). Пользователь может создавать граф вручную или генерировать случайный граф заданного размера, запускать алгоритм и наблюдать пошаговую визуализацию его работы. Приложение должно уверенно работать с графами размером от 70 до 90 вершин, обеспечивая читаемое отображение структуры графа и удобную навигацию.

Функциональные возможности 2.1. Создание графа вручную Пользователь может добавлять вершины кликом мыши по холсту. Вершины отображаются в виде кружков с порядковым номером внутри. Поддерживается автоматический режим: клик по пустому месту создаёт вершину, клик по существующей вершине начинает создание ребра. Также доступны раздельные режимы: только вершины или только рёбра, переключаемые кнопками на панели инструментов.

Пользователь может добавлять ориентированные рёбра. Для этого нужно кликнуть по первой вершине (она подсвечивается жёлтым), затем по второй. Появляется диалоговое окно для ввода веса ребра. Ребро отображается в виде линии со стрелкой на конце и весом посередине. При повторном проведении ребра между теми же вершинами его вес обновляется. Если существует обратное ребро, его вес автоматически синхронизируется. При создании встречного ребра вес подставляется автоматически без диалога.

Кнопка "Очистить" полностью удаляет граф. Кнопка "Расставить" автоматически перестраивает расположение вершин для оптимального отображения.

2.2. Автоматическая генерация графа Пользователь может сгенерировать случайный граф, указав желаемое количество вершин. Генератор создаёт указанное количество вершин и случайное количество рёбер между ними со случайными весами. Вершины автоматически размещаются с использованием силового алгоритма раскладки (force-directed layout), который обеспечивает равномерное распределение вершин по холсту без наложений, группировку связанных вершин рядом друг с другом, минимизацию пересечений рёбер и читаемость структуры графа.

Параметры генерации: количество вершин от 5 до 100, вероятность создания ребра между двумя вершинами (по умолчанию 15%), диапазон весов рёбер (по умолчанию от 1 до 20).

После генерации пользователь может редактировать граф вручную: добавлять и удалять вершины и рёбра, перетаскивать вершины, менять веса рёбер.

2.3. Автоматическая раскладка графа Алгоритм force-directed layout работает по следующему принципу: каждая вершина отталкивается от всех остальных как одноимённый заряд, а рёбра стягивают связанные вершины как пружины. Процесс итеративный: на каждой итерации вычисляются силы, действующие на каждую вершину, и вершины сдвигаются под их действием. Количество итераций зависит от размера графа: 100 итераций для графов до 30 вершин, 200 итераций для графов от 30 до 70 вершин, 300 итераций для графов больше 70 вершин.

Раскладка запускается автоматически после генерации графа. Также её можно запустить вручную кнопкой "Расставить" для уже существующего графа. После раскладки вершины получают новые координаты, и холст перерисовывается.

2.4. Навигация по холсту Холст можно масштабировать колёсиком мыши с диапазоном зума от 0.1x до 5.0x. Зум работает относительно положения курсора: точка под курсором остаётся на месте, остальное масштабируется. Холст можно перетаскивать, зажав правую кнопку мыши. Координаты вершин корректно пересчитываются с учётом текущего масштаба и смещения.

При масштабировании меняется уровень детализации. При мелком масштабе (зум меньше 0.5x) радиус вершин уменьшается до 10 пикселей, номера внутри вершин скрываются, толщина линий рёбер уменьшается. При крупном масштабе (зум больше 1.5x) радиус увеличивается до 30 пикселей, номера отображаются крупным шрифтом, веса рёбер отображаются читаемо. При среднем масштабе используются стандартные настройки: радиус 20 пикселей, обычный шрифт.

Перетаскивание отдельных вершин мышью позволяет пользователю вручную скорректировать расположение после автоматической раскладки.

2.5. Запуск алгоритма Пользователь выбирает стартовую и конечную вершины. Для этого используются кнопки "Выбрать старт" и "Выбрать финиш" на панели инструментов, либо двойной клик по вершине: первый двойной клик назначает вершину стартовой, второй — конечной. Стартовая вершина подсвечивается зелёным, конечная — красным.

После выбора обеих вершин кнопка "Запустить" становится активной. При нажатии запускается алгоритм Форда-Беллмана, результат сохраняется.

2.6. Пошаговая визуализация После запуска алгоритма пользователь может просматривать его работу по шагам. Кнопка "Шаг вперёд" показывает очередной шаг. Визуальные обозначения шага: текущая обрабатываемая вершина подсвечивается жёлтым, рассматриваемое ребро — оранжевым, посещённые вершины — серым цветом, стартовая и конечная сохраняют свои цвета. Рядом с вершинами отображаются текущие расстояния. В информационной панели внизу выводится текстовое пояснение происходящего на текущем шаге.

Предусмотрен автоматический режим проигрывания шагов с задержкой 500 миллисекунд между шагами, запускаемый кнопкой "Авто". Кнопка "Сброс" возвращает визуализацию в начальное состояние, убирает всю подсветку.

Итоговый кратчайший путь отображается красной линией увеличенной толщины. Общая длина пути выводится в информационной панели.

Алгоритм Форда-Беллмана умеет обнаруживать отрицательные циклы. При обнаружении отрицательного цикла выводится сообщение "Обнаружен отрицательный цикл. Кратчайший путь не существует.", и визуализация останавливается.

2.7. Сохранение и загрузка графа Граф можно сохранить в текстовый файл с расширением .graph и загрузить из файла. Формат файла: сначала секция VERTICES со строками вида "id name", затем секция EDGES со строками вида "from to weight". Координаты вершин не сохраняются, при загрузке применяется автоматическая раскладка.

При загрузке графа из файла или генерации случайного графа, если количество вершин превышает 30, радиус вершин и размер шрифта автоматически уменьшаются для предотвращения наложений.

2.8. Очистка и сброс Кнопка "Сброс" отменяет результат алгоритма и возвращает граф в исходное состояние: убирает всю подсветку, очищает историю шагов, оставляет вершины и рёбра без изменений. Кнопка "Очистить" удаляет граф полностью, включая все вершины и рёбра.

Технические требования Язык программирования: Java, JDK версии не выше 21. Сборка через Gradle с плагином application, результат — исполняемый jar-файл. Приложение должно работать на системах без установленной среды разработки.
Графический интерфейс реализован на Swing. Главное окно разделено на три области: холст для рисования графа в центре, панель инструментов с кнопками справа, информационная панель с текстовыми пояснениями внизу. Холст поддерживает двойную буферизацию для плавной отрисовки.

Модель графа использует список смежности (Map<Vertex, List>) для быстрого доступа к соседям вершины. Граф ориентированный. Метод getNeighbors(v) возвращает только исходящие рёбра. Для алгоритма Форда-Беллмана строится полный список рёбер проходом по всем вершинам и сбором всех исходящих рёбер.

Алгоритм force-directed layout реализован в классе Graph или отдельном классе-утителе. На каждой итерации для каждой пары вершин вычисляется сила отталкивания (кулоновская, обратно пропорциональная квадрату расстояния), а для каждого ребра — сила притяжения (гуковская, пропорциональная разнице между текущей длиной и желаемой). Вершины сдвигаются на величину, пропорциональную равнодействующей силе, с коэффициентом затухания.

Ограничения Вершины нумеруются автоматически с нуля, идентификаторы уникальны. Вес ребра — целое число, может быть отрицательным. Количество вершин ограничено 100 для ручного режима, для автоматической генерации ограничений нет в пределах разумного. Приложение однопользовательское, сетевого взаимодействия нет. Отрицательные циклы обрабатываются с выводом сообщения об ошибке.
